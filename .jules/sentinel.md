## 2024-05-24 - Unescaped backslashes in SQLite LIKE queries
**Vulnerability:** User inputs were not properly escaping the backslash `\` character when used in SQLite `LIKE` queries with the `ESCAPE '\'` clause, which could lead to malformed SQL if a user searches for a literal backslash. In JavaScript template literals, the string `"\\"` is translated to `\` in SQL. We need to replace it with `"\\\\"` in Javascript strings so it produces `\\` in SQL, which is the escaped backslash character. Also, in `routes/logs.ts`, `ESCAPE '\'` was missing entirely for some LIKE parameters.
**Learning:** `ESCAPE '\'` requires exactly one literal backslash character in the SQL expression. So the SQL string must be `ESCAPE '\'`, which in JS is written as `ESCAPE '\\'`. When a user searches for `\`, the JS string must be replaced to contain `\\`, which translates to `%\\%` in SQL. The regex `/[%_\\]/g` must replace all `_`, `%` and `\` with `\x` (where x is the matched char). In JS, this is `.replace(/[%_\\]/g, '\\$&')`.
**Prevention:** Always escape user input strings for `LIKE` using `.replace(/[%_\\]/g, '\\$&')` and always include the `ESCAPE '\\'` clause on all `LIKE` conditions.
