## 2024-05-24 - Fix Wildcard Injection in SQLite LIKE Queries
**Vulnerability:** Unescaped wildcards (`%`, `_`) and unescaped backslashes (`\`) in user input used in SQLite `LIKE` queries allowed users to perform wildcard injection/DoS attacks. In Node.js, `ESCAPE '\'` in a template literal translates to `ESCAPE ''` (which is invalid syntax) rather than `ESCAPE '\'`.
**Learning:** `LIKE` queries using string concatenation or partial parameterization are vulnerable if special characters are not thoroughly escaped. Additionally, when defining the `ESCAPE` clause in Javascript template literals for `better-sqlite3`, you must use `ESCAPE '\\'` (two backslashes). In Javascript template strings, `\\` evaluates to a single literal backslash `\`. SQLite expects a *single* character for the `ESCAPE` expression. If you use `ESCAPE '\\\\'`, SQLite receives `\\` (two literal backslashes) and throws `Error: ESCAPE expression must be a single character` during query preparation.
**Prevention:** Always escape `%`, `_`, and `\` using `.replace(/[\\%_]/g, '\\$&')`. Always cast user input with `String()` before replacing. Always use `ESCAPE '\\'` in Javascript string literals so that SQLite correctly parses it as `ESCAPE '\'`.
